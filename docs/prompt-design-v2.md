# CleanCare Prompt 体系 v2

## 1. 目标

Prompt v2 面向中文 OpenAI-compatible 模型，服务于清洁电器领域的意图识别、查询改写、规划、回答生成、反思、澄清、会话摘要和评测。

## 2. 统一契约

每个 Prompt 必须包含：

1. 身份
2. 任务
3. 业务背景
4. 输出标准或模板
5. 约束规则
6. 至少两个完整 Few-Shot 示例
7. 输出前自检

契约由 `internal/prompt/prompt_test.go` 自动检查，避免后续修改退化为只有一句指令的模板。

## 3. 设计原则

- 数值必须与证据完全一致，不能使用“大约”“一般”等词修饰未提供的参数。
- 静态规格引用知识库证据；价格、库存、订单和保修引用本轮工具结果。
- 对比必须保证每个型号有独立证据，缺少同口径数据时明确写“无法比较”。
- 故障诊断先检查安全红线，每轮只推进一个高区分度问题。
- 售后结论使用条件句，不输出无条件的“能退”或“不能退”。
- 工具名、内部路径、模型配置和推理过程不出现在面向用户的回答中。

## 4. 模板清单

代码当前注册 12 个场景：`system`、`intent`、`rewrite`、`plan`、`generate_generic`、`generate_compare`、`generate_diagnose`、`generate_policy`、`reflect`、`clarify`、`summarize`、`eval_judge`。

用户需求中称“11 个模板”，但现有代码实际有 12 个可注册场景。本次按代码真实数量全部升级，未删除摘要或评测模板。

## 5. 动态注入

模板只描述稳定的领域规则。用户问题、会话摘要、实体、证据、工具结果、诊断状态和评测上下文通过占位符注入。动态价格、库存和政策生效状态不得写死在系统 Prompt。

## 6. 版本与回滚

所有默认模板版本为 `v2`。Registry 保留版本历史并支持激活旧版本；调用链在 trace 中记录场景和版本，便于评测回归与问题定位。
